# Инициализация пакетов

В Go инициализация переменных на уровне пакета происходит не сверху вниз (как во многих скриптовых языках), а на основе анализа зависимостей (Dependency Analysis).

```Golang
var a = b + c 
var b = f()
var c = 1

func f() int { return c + 1 }
```
В данном примере:
1. Первой инициализируется переменная ```c``` со значением 1, так как она является константным выражением и не имеет внешних зависимостей.

2. Второй инициализируется переменная ```b```. Компилятор вычисляет её значение путем вызова функции ```f()```, которая на момент вызова уже может безопасно использовать ранее инициализированную переменную ```c```. В результате переменная ```b``` получает значение 2.

3. Третьей инициализируется переменная ```a```. На этом этапе все её зависимости (```b``` и ```c```) уже полностью разрешены, поэтому компилятор успешно вычисляет и присваивает ей итоговое значение 3.

Если пакет состоит из нескольких исходных файлов .go, то они инициализируются в том порядке, в каком файлы передались компилятору.

В случаях, когда инициализирующее выражение является не самым простым средством задания начального значения - используется функция ```init()```. Любой файл .go может содержать любое количество таких функций.

```Golang
func init() { /* ... */ }
```
Функцию ```init()``` нельзя вызвать или обратиться к ней. В файле данная функция выполняется автоматически при запуске программы в том порядке, в котором они объявлены.
