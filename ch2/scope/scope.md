# Область видимости

**Область видимости** — это область исходного текста программы, в которой объявленное имя (переменная, функция, тип) можно использовать. Это свойство времени компиляции.

**Синтаксический блок** — последовательность инструкций, явно заключенная в фигурные скобки ```{ }``` (например, тело функции или цикла).

**Лексический блок** — более широкое понятие. Это группа объявлений, которая определяет область видимости, даже если она не выделена фигурными скобками явно (например, весь файл или пакет).

## Иерархия лексических блоков
1. **Всеобщий блок (Universe block)** — охватывает вообще весь исходный код. Содержит встроенные типы (int, string), константы (true, false) и функции (len, make, append).

2. **Блок пакета (Package block)** — охватывает все файлы одного пакета. Объявления на уровне пакета видны во всех файлах этого пакета.

3. **Блок файла (File block)** — охватывает только один конкретный файл. Здесь живут, например, импорты (import "fmt").

4. **Локальные блоки (Local blocks)**:

    * Блок функции (тело функции).

    * Блоки инструкций for, if, switch.

    * Блок каждого case в инструкциях switch или select.

    * Явные синтаксические блоки { } внутри кода.

## Поиск имён компилятором (Правило матрёшки)

Компилятор ищет объявление имени изнутри наружу: от самого глубокого локального блока до всеобщего. Если имя не найдено нигде — генерируется ошибка undeclared name.

## Затенение переменных (Shadowing)
            x := x + 'A' - 'a'      // x №3 (Локальная для тела блока if). Затеняет x №2.

Если имя объявлено и во внешнем, и во внутреннем блоках, компилятор первым обнаружит внутреннее объявление. Оно затеняет (скрывает) внешнее.

```Golang
var g = "g"
func f() {}

func main() {
    f := "f"       // Локальная переменная f затеняет 
                   // функцию f уровня пакета
    fmt.Println(f) // Выведет: "f"
    fmt.Println(g) // Выведет: "g" (переменная уровня пакета)
    fmt.Println(h) // Ошибка компиляции: неопределенное имя h
}
```

Инструкции if, for, switch создают два лексических блока:

1. Неявный (внешний): охватывает переменные, созданные в блоке инициализации/условия.

2. Явный (внутренний): тело конструкции внутри фигурных скобок { }.

## Множественное затенение в циклах for
Внутри функции блоки могут быть вложены на любую глубину. Пример ниже содержит 3 разных переменных ```x```:

```Golang
func main() {
    x := "hello!"
    for i := 0; i < len(x); i++ {
        x := x[i]
        if x != '!' {
            x := x + 'A' - 'a'
            fmt.Printf("%c", x) // "HELLO" (по букве за итерацию)
        }
    }
}
```
## Цепочки if - else if - else
Каждый следующий if вложен в предыдущий. Переменные из инициализатора первого if видны во всей цепочке, но не за её пределами.

```Golang
if x := f(); x == 0 {
    fmt.Println(x)
} else if y := g(x); x == y {
    fmt.Println(x, y)
} else {
    fmt.Println(x, y)
}
fmt.Println(x, y)        // Ошибка компиляции: x и y здесь невидимы!
```

## Ошибки проектирования областей видимости
Если переменная нужна после блока if, её объявляют до него.

1. Плохой стиль (излишний отступ): Пытаться запихнуть весь код в else, чтобы сохранить область видимости переменной f:
```Go
if f, err := os.Open(fname); err != nil {
    return err
} else {
    f.Stat() // f и err видны, но код смещается вправо
    f.Close()
}
```
2. Идиоматичный стиль Go: Обработать ошибку и сразу выйти, оставив успешный путь выполнения без лишних отступов.

```Golang
f, err := os.Open(fname)
if err != nil {
    return err
}
f.Stat()
f.Close()
```
## Короткое объявление := и init()

Когда используется := для нескольких переменных, где одна уже создана, а вторая — новая, Go может случайно создать две локальные переменные, затенив глобальную.

Пример скрытого бага:

```Go
var cwd string // Глобальная переменная уровня пакета

func init() {
    cwd, err := os.Getwd() // БАГ! := создал локальную cwd, так как err — новая переменная.
    if err != nil {
        log.Fatalf("Ошибка os.Getwd: %v", err)
    }
    log.Printf("Рабочий каталог = %s", cwd) // Выведет путь, но запишет его в локальную cwd.
}

func main() {
    // Здесь глобальная cwd осталась пустой, так как init() её не обновил.
}
```

Нужно явно объявить переменную ошибки через var, чтобы использовать обычное присваивание = для глобальной переменной:

```Go
var cwd string 

func init() {
    var err error // Явное объявление
    cwd, err = os.Getwd() // Используется =. 
    if err != nil {
        log.Fatalf("Ошибка os.Getwd failed: %v", err)
    }
}
```
