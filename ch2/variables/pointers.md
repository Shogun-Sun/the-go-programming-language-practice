# Указатели
Указатель - это переменная, значением которой является адрес другой переменной в памяти. ***Не всякое значение имеет адрес, но его имеет любая переменная***. То-есть данный код выдаст ошибку при компиляции:
```Golang
p := &42
p2 := &(x + 1)
```

С помощью указателя можно считывать или изменять значение переменной косвенно, не используя ее имя.
Если переменная объявлена как ```var x int```, то выражение ```&x```("адрес x") дает указатель на целочисленную переменную - ```*int``` ("указатель на int"). Если ```p := &x```, то говорится: "p указывает на x"

Переменная, на которую указывает p, записывается как ```*p``` - и дает значение этой переменной. Такая запись может использоваться и в левой части, в таком случае присваение обновляет данную переменную. 

```Golang
x := 1
p := &x // Имеет тип *int и указывает на x
fmt.Println(*p) // 1
*p = 2 // x = 2
fmt.Println(*p) // 2 
```

***Нулевое значение указателя любого типа равно nil.*** Тогда проверка ```p != nil``` истинна если p указывает на переменную. 

Указатели можно сравнивать. *Два указателя равны тогда и только тогда, когда они указывают на одну и ту же переменную или когда они оба равны nil.*
```Golang
var x, y int
fmt.Println(&x == &x, &x == &y, &x == nil) // true, false, false
```

Функция может спокойно вернуть адрес локальной переменной
```Golang
var p = f()

func f() *int {
    v := 1
    return &v
}
```
В данном примере локальная переменная ```v``` созданная вызовом ```f()``` продолжит существовать после выхода из функции.

Все вызовы ```f()``` будут возвращать различные значения:
```Golang
fmt.Println(f() == f()) // false
```

При получении адреса переменной или копируя указатель - создаются новые псевдонимы (способы идентификации одной переменной) ```*p```. Псевдонимы указатели позволяют получить доступ к переменной без использовании ее имени.
