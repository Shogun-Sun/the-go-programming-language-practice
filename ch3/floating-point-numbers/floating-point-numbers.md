# Числа с плавающей точкой (Floating-Point Numbers)

В Go представлены два типа чисел с плавающей точкой. Их свойства строго регулируются международным стандартом **IEEE 754** и аппаратно ускоряются всеми современными процессорами.

## `float32` и `float64`

| Тип данных | Точность (значащие цифры) | Максимальное значение | Минимальное положительное значение |
| --- | --- | --- | --- |
| **`float32`** | ~6 десятичных цифр | `math.MaxFloat32` ($\approx 3.4 \times 10^{38}$) | $\approx 1.4 \times 10^{-45}$ |
| **`float64`** | ~15 десятичных цифр | `math.MaxFloat64` ($\approx 1.8 \times 10^{308}$) | `math.SmallestNonzeroFloat64` ($\approx 4.9 \times 10^{-324}$) |

> В большинстве случаев следует предпочитать тип **`float64`**, так как вычисления с `float32` слишком быстро накапливают ошибку округления.

---

## Ловушка точности (Precision Trap)

Из-за фиксированного размера памяти под мантиссу числа, `float32` не может точно представить крупные целые числа.

```go
var f float32 = 16777216    // 1 << 24
fmt.Println(f == f+1)       // true
```

*Почему так происходит?* Числу `float32` просто не хватает битов точности, чтобы заметить прибавление единицы на таком порядке величины.

---

## Варианты записи литералов

Числа с плавающей точкой можно объявлять несколькими способами:

* **Стандартный десятичный:** `const e = 2.71828`
* **С сокращением нуля:** `.707` (вместо `0.707`) или `1.` (вместо `1.0`)
* **Научный (экспоненциальный) формат** — идеален для очень больших или очень малых чисел (использует букву `e` или `E`):
```go
const Avogadro = 6.02214129e23
const Planck   = 6.62606957e-34

```

---

## Управление выводом в `fmt.Printf`

Для печати чисел с плавающей точкой используются три основных флага:

* **`%g`** — автоматически выбирает самое компактное представление числа (сворачивает в `e`, если число огромное). Используется по умолчанию.
* **`%f`** — выводит число как стандартную десятичную дробь (без экспонент). Удобно для таблиц.
* **`%e`** — принудительно выводит число в научном формате (с экспонентой).

#### Спецификаторы ширины и точности:

Выражение вида `%[ширина_поля].[точность][флаг]` позволяет форматировать текст:

```go
package main

import (
	"fmt"
	"math"
)

func main() {
	for x := 0; x < 8; x++ {
		// %8.3f — поле шириной 8 символов, ровно 3 знака после запятой
		fmt.Printf("x = %d e^x = %8.3f\n", x, math.Exp(float64(x)))
	}
}

```

**Вывод программы в консоль:**

```text
x = 0 e^x =    1.000
x = 1 e^x =    2.718
x = 2 e^x =    7.389
x = 3 e^x =   20.086
x = 4 e^x =   54.598
x = 5 e^x =  148.413
x = 6 e^x =  403.429
x = 7 e^x = 1096.633

```

## Специальные числовые значения (Inf и NaN)

Стандарт IEEE 754 определяет особые состояния для результатов ошибочных математических операций.

* **`+Inf` и `-Inf**` — бесконечность. Возникает при делении ненулевого числа на ноль или при выходе за пределы максимума.
* **`NaN` (Not a Number)** — не число. Возникает при неопределенностях вроде `0/0` или извлечении корня из отрицательного числа `math.Sqrt(-1)`.

```go
var z float64
fmt.Println(z, -z, 1/z, -1/z, z/z) 
// Выведет: "0 -0 +Inf -Inf NaN"

```

> Не стоит пытаться проверить, равен ли результат переменной `NaN` через обычное сравнение (`==`). По стандарту **`NaN` не равен вообще ничему, даже самому себе**.

```go
nan := math.NaN()
fmt.Println(nan == nan, nan < nan, nan > nan) 
// Выведет: "false false false"

```

**Правильная проверка:**
Для обнаружения этого значения используйте специальную встроенную функцию из пакета `math`:

```go
if math.IsNaN(yourFloatVariable) {
    // Обработка ошибки «Не число»
}

```
